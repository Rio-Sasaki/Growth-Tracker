# RLSポリシー設計

## 概要
全テーブルにRow Level Security（RLS）を設定し、他ユーザーのデータへのアクセスを禁止しています。

## ポリシー一覧

| テーブル | SELECT | INSERT | UPDATE | DELETE |
|---|---|---|---|---|
| `profiles` | 自分のみ | 自分のみ | 自分のみ（with check） | 自分のみ |
| `categories` | 全ユーザー | 不可 | 不可 | 不可 |
| `books` | 全ユーザー | 全ユーザー | 不可 | 不可 |
| `user_books` | 自分のみ | 自分のみ | 自分のみ（with check） | 自分のみ |
| `memos` | 自分のみ | 自分のみ | 自分のみ（with check） | 自分のみ |
| `tags` | 自分のみ | 自分のみ | 自分のみ（with check） | 自分のみ |
| `memo_tags` | 自分のみ | 自分のみ | 不可 | 自分のみ |
| `studies` | 自分のみ | 自分のみ | 自分のみ（with check） | 自分のみ |

## 設計上の決定事項

- **`categories` は全ユーザー参照のみ**：システム固定のマスタデータのため、ユーザーによる変更は不可。
- **`books` は全ユーザー参照・作成可能、更新・削除は不可**：書籍マスタは共有リソースのため、誰かが勝手に書き換えられないようにする。
- **UPDATEには `with check` を追加**：更新後のデータが自分のものであることを保証する。
- **`memo_tags` はUPDATEなし**：タグの付け外しは削除・作成で対応するため更新は不要。
